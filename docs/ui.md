# 이미지 뷰어 UI 문서

## 1. 문서 목적

이 문서는 이미지 뷰어의 UI 방향, 화면 구성, 상호작용 원칙을 정의한다.
기준점은 꿀뷰와 유사한 빠른 이미지 탐색 경험이지만, macOS 환경에 더 자연스러운 방식으로 재구성하는 것을 목표로 한다.

## 2. UI 목표

앱의 UI는 다음 원칙을 따른다.

- 이미지가 항상 화면의 주인공이어야 한다.
- 자주 쓰는 기능은 빠르게 접근 가능해야 한다.
- 자주 쓰지 않는 기능은 화면을 점유하지 않아야 한다.
- 키보드, 마우스, 트랙패드 중심 사용 흐름을 지원해야 한다.
- 꿀뷰처럼 빠르게 쓸 수 있어야 하지만 macOS 앱처럼 단정하고 가벼워야 한다.

## 3. 디자인 방향

### 3.1 기본 방향

권장 UI 방향은 "미니멀 캔버스형 뷰어"이다.

- 중앙에는 이미지만 크게 표시한다.
- 상시 노출되는 조작 버튼은 최소화한다.
- 필요한 정보와 컨트롤은 오버레이로 잠시 나타나고 사라진다.
- 메뉴바, 단축키, 툴바를 적극 활용한다.

### 3.2 꿀뷰와의 차별점

꿀뷰는 기능이 많고 빠르지만, Windows 스타일의 고정형 UI 감각이 있다.
본 프로젝트는 다음 점에서 더 단순하고 현대적인 방향을 택한다.

- 상시 노출 버튼 대신 자동 숨김 오버레이 사용
- 기능 버튼 나열 대신 보기 메뉴와 컨텍스트 메뉴 활용
- 상태 정보는 필요 시 토글 방식으로 표시
- 이미지 감상 중 시야를 가리지 않는 얇은 툴바 구성

## 4. 전체 화면 구조

기본 화면은 다음 4개 영역으로 구성한다.

- 메인 캔버스
- 상단 툴바
- 하단 상태 오버레이
- 임시 옵션 패널
- 파일 목록 브라우저
- OCR/번역 결과 패널

### 4.1 메인 캔버스

메인 캔버스는 이미지가 실제로 표시되는 주 영역이다.

- 창의 대부분을 차지한다.
- 배경은 중립적인 단색 또는 아주 약한 질감으로 구성한다.
- 기본 배경은 어두운 회색 계열을 권장한다.
- 이미지가 없을 때는 빈 상태 화면을 표시한다.

### 4.2 상단 툴바

상단 툴바는 자주 쓰는 조작만 최소한으로 제공한다.

권장 항목:

- 파일 열기
- 최근 파일 열기
- 이전 이미지
- 다음 이미지
- 현재 컬렉션 목록 보기
- 보기 모드 선택
- 확대/축소
- 전체화면

세부 원칙:

- 텍스트보다 아이콘 중심으로 구성한다.
- 아이콘 수를 과도하게 늘리지 않는다.
- 회전, 반전, 스케일 방식 같은 항목은 기본 툴바가 아니라 메뉴 또는 패널로 보낸다.
- 현재 구현에서는 macOS 기본 툴바의 자동 숨김이 불안정해 상단 툴바는 상시 표시를 기본 정책으로 둔다.

### 4.3 하단 상태 오버레이

하단 상태 오버레이는 현재 보고 있는 이미지의 핵심 정보를 짧게 보여준다.

표시 후보:

- 파일명
- 현재 순번과 전체 개수
- 해상도
- 현재 배율
- 현재 보기 모드

세부 원칙:

- 반투명 배경 위에 한 줄 또는 두 줄 수준으로 배치한다.
- 항상 노출하지 않고 마우스 이동 시 또는 토글 시 노출한다.
- 정보량이 많아지면 정보 패널과 분리한다.
- 현재 구현에서는 하단 상태 오버레이만 자동 숨김 대상으로 사용한다.

### 4.4 임시 옵션 패널

임시 옵션 패널은 현재 이미지의 보기 관련 설정을 바꾸는 용도로 사용한다.

포함 후보:

- 화면 맞춤 방식
- 스케일 방식
- 회전
- 좌우 반전

표시 방식:

- 툴바 버튼 클릭 시 팝오버
- 우클릭 컨텍스트 메뉴
- 메뉴바의 View 메뉴

### 4.5 파일 목록 브라우저

파일 목록 브라우저는 현재 탐색 중인 컬렉션을 한눈에 보고 바로 점프하는 용도로 사용한다.

- 일반 이미지 열기 상태에서는 같은 폴더 이미지 목록을 보여준다.
- ZIP 열기 상태에서는 압축 내부 이미지 목록을 보여준다.
- 현재 보고 있는 항목은 목록에서 시각적으로 강조한다.
- 기본 노출 방식은 시트 또는 패널이다.

### 4.6 OCR/번역 결과 패널

OCR과 번역은 이미지 감상 흐름을 끊지 않는 보조 기능으로 다룬다.

- 기본 노출 방식은 우측 사이드 패널 또는 시트이다.
- 패널에는 원문, 번역문, 대상 언어, 복사 액션을 포함한다.
- OCR 실행 중에는 짧은 진행 상태를 표시한다.
- 텍스트가 없으면 빈 결과가 아니라 "감지된 텍스트 없음" 상태를 보여준다.
- 기본 정책은 이미지 위에 텍스트를 덮어쓰지 않고 결과 패널로 먼저 제공하는 것이다.

## 5. 화면 상태

### 5.1 빈 상태

앱 실행 직후 또는 파일이 없는 상태에서 보이는 화면이다.

구성 요소:

- 중앙 드롭존
- "이미지 또는 ZIP 파일을 여기에 놓기" 안내
- 파일 열기 버튼
- 지원 포맷 간단 안내

원칙:

- 장식보다 기능 안내를 우선한다.
- 첫 실행 사용자가 바로 무엇을 해야 하는지 알아야 한다.

### 5.2 일반 보기 상태

이미지가 열려 있고 별도 조작이 없는 기본 상태이다.

- 메인 캔버스가 화면 대부분을 차지한다.
- 상단 툴바는 얇게 유지한다.
- 상태 오버레이는 자동 숨김 가능해야 한다.

### 5.3 조작 중 상태

확대/축소, 패닝, 보기 모드 변경 등 사용자가 조작 중일 때의 상태이다.

- 조작 결과가 즉시 화면에 반영되어야 한다.
- 배율이나 보기 모드 변경 시 짧은 HUD 형태의 피드백을 표시할 수 있다.
- 조작이 끝나면 UI는 다시 조용한 상태로 돌아가야 한다.

### 5.4 전체화면 상태

전체화면 상태는 몰입형 감상용 모드이다.

- 창 크롬을 최소화한다.
- 툴바는 숨기거나 자동 숨김 처리한다.
- 정보 오버레이와 탐색 동작은 그대로 유지한다.
- OCR/번역 패널은 필요 시 슬라이드 인 형태로 열고 닫을 수 있어야 한다.

### 5.5 텍스트 분석 상태

사용자가 OCR 또는 번역을 실행한 직후의 상태이다.

- 이미지 자체의 위치와 배율은 유지한다.
- 결과는 별도 패널에서 업데이트한다.
- 긴 텍스트도 읽을 수 있게 스크롤 가능한 영역을 둔다.
- 실패 상태와 성공 상태를 시각적으로 구분한다.

## 6. 상호작용 원칙

### 6.1 키보드 우선

핵심 탐색은 키보드만으로 가능해야 한다.

- 좌/우 방향키로 이동
- Home/End로 처음/마지막 이동
- `0`, `1` 등으로 보기 모드 복귀
- `Tab` 또는 별도 키로 정보 오버레이 토글

### 6.2 마우스 우선 보조

마우스는 직관적인 보조 입력 역할을 한다.

- 휠: 이전/다음 이미지 이동
- 수정 키 + 휠: 확대/축소
- 드래그: 확대 상태에서 패닝
- 더블클릭: 맞춤/100% 전환 또는 전체화면 전환

### 6.3 트랙패드 친화성

macOS 환경에서는 트랙패드 사용 비율이 높으므로 다음을 고려한다.

- 스와이프로 이전/다음 이미지 이동
- 핀치로 확대/축소
- 두 손가락 이동과 이미지 넘김 정책 충돌 방지

1차 구현에서는 트랙패드 전용 제스처를 선택 항목으로 두고, 키보드와 마우스를 우선 구현한다.

## 7. 정보 노출 정책

UI는 "항상 보이기"보다 "필요할 때 보이기"를 기본으로 한다.

### 7.1 항상 보여도 되는 것

- 상단의 최소 툴바
- 메인 이미지

### 7.2 기본 숨김이 좋은 것

- 파일명과 순번
- 해상도와 배율
- 보기 모드/스케일 방식 상태
- 회전 및 반전 버튼
- OCR/번역 결과 패널

현재 구현 기준:

- 상단 툴바: 항상 표시
- 하단 상태 오버레이: 자동 숨김
- 현재 컬렉션 목록: 툴바 버튼으로 시트 표시

### 7.3 표시 트리거

다음 경우에 오버레이를 노출할 수 있다.

- 마우스 이동
- 키보드 조작 직후
- 보기 모드 변경 직후
- 사용자가 명시적으로 정보 표시를 켠 경우

## 10. 시각 스타일 가이드

### 10.1 톤

- 무광택 느낌의 차분한 다크 UI를 기본값으로 한다.
- 색은 절제하고, 기능 강조에만 제한적으로 사용한다.
- 배경은 순수 검정보다 약간 밝은 중성 회색이 안정적이다.

### 10.2 타이포그래피

- 시스템 폰트를 사용한다.
- 상태 정보는 작은 크기지만 읽기 쉬운 대비를 유지한다.
- 정보 계층은 크기보다 굵기와 투명도로 구분한다.

### 10.3 모서리와 그림자

- 오버레이는 약한 블러 또는 반투명 배경을 사용한다.
- 모서리는 과하게 둥글지 않게 한다.
- 그림자는 최소화한다.

## 11. 구현 우선순위

### 11.1 1차 UI 구현

- 빈 상태 화면
- 메인 이미지 캔버스
- 상단 최소 툴바
- 하단 상태 오버레이
- 전체화면 전환

### 11.2 2차 UI 구현

- 옵션 팝오버
- 컨텍스트 메뉴
- 회전/반전 HUD
- 최근 파일 메뉴 정리
- 파일 목록 브라우저 UX 정리
- OCR 결과 패널
- 번역 결과 패널

### 11.3 후속 검토

- 썸네일 스트립
- 사이드 정보 패널
- 멀티 윈도우 UX
- 웹툰/세로 스크롤 모드 전용 UI
- 이미지 위 텍스트 영역 하이라이트

## 12. 결정 사항

현재 UI 방향은 다음으로 정리한다.

- 꿀뷰식 빠른 탐색 경험은 유지한다.
- 고정형 복잡 UI 대신 미니멀 캔버스형 구조를 채택한다.
- 키보드와 마우스를 주 입력으로 삼는다.
- 정보와 옵션은 오버레이 중심으로 제공한다.
- macOS 기본 메뉴, 툴바, 전체화면 패턴을 적극 활용한다.
- OCR과 번역은 메인 캔버스를 침범하지 않는 보조 패널로 제공한다.
