# 이미지 뷰어 구현 순서 문서

## 1. 문서 목적

이 문서는 현재 정의된 요구사항과 UI 문서를 바탕으로, 실제 구현을 어떤 순서로 진행할지 정리한 개발 계획서이다.
목표는 빠르게 동작하는 MVP를 먼저 만들고, 그 위에 보기 옵션과 압축 파일 탐색, UI 개선을 단계적으로 올리는 것이다.

## 2. 구현 원칙

- 가장 먼저 "이미지 하나를 안정적으로 연다"를 만든다.
- 그 다음 "여러 이미지를 순서대로 넘긴다"를 만든다.
- 그 다음 "보는 방식과 UI를 다듬는다".
- 압축 파일 지원은 일반 파일 탐색 구조가 안정된 뒤에 붙인다.
- OCR/번역은 핵심 뷰어 흐름이 안정된 뒤 후반부에 붙인다.
- 처음부터 모든 기능을 넣지 않고, 단계마다 실행 가능한 상태를 유지한다.

## 3. 전체 단계 요약

### Phase 1. 앱 골격과 기본 열기

목표:
앱을 실행하고 이미지 파일 하나를 열어 화면에 표시할 수 있게 만든다.

### Phase 2. 이미지 탐색 모델

목표:
같은 폴더 안의 이미지 목록을 구성하고 이전/다음 이동이 되게 만든다.

### Phase 3. 기본 뷰어 UI

목표:
캔버스 중심 UI, 툴바, 상태 오버레이, 빈 상태 화면을 만든다.

### Phase 4. 보기 제어

목표:
확대/축소, 맞춤 방식, 스케일 방식, 회전/반전을 붙인다.

### Phase 5. 상호작용 정리

목표:
키보드, 마우스, 드래그 동작, 자동 숨김 UI를 정리한다.

### Phase 6. 압축 파일 지원

목표:
ZIP 파일 내부 이미지를 일반 폴더처럼 탐색하게 만든다.

### Phase 7. 마무리 및 안정화

목표:
오류 처리, 최근 파일, 상태 저장, 성능 개선을 진행한다.

### Phase 8. OCR 및 온디바이스 번역

목표:
현재 이미지에서 텍스트를 인식하고, Apple 제공 온디바이스 번역으로 결과를 확인하게 만든다.

## 4. 단계별 상세 계획

### 4.1 Phase 1. 앱 골격과 기본 열기

이 단계는 가장 작고 확실한 동작 단위를 만드는 단계이다.

구현 항목:

- 메인 윈도우에 단일 `ViewerScreen` 배치
- 파일 열기 액션 추가
- Finder에서 전달된 파일 URL 수신 처리
- 단일 이미지 로딩 및 화면 표시
- 이미지가 없을 때 빈 상태 화면 표시

권장 작업 순서:

1. `App` 진입점과 메인 뷰 구조 정리
2. 파일 URL을 앱 상태로 전달하는 경로 만들기
3. 이미지 로더 구현
4. 로딩 성공/실패/빈 상태 분기

완료 기준:

- 앱 실행 후 `Open...`으로 이미지 파일을 열 수 있다.
- Finder에서 앱으로 파일 열기 요청이 오면 표시된다.
- 파일이 없을 때 빈 상태 화면이 보인다.

현재 상태:

3. 정렬 규칙 구현
4. 현재 파일 위치 계산
5. 이전/다음 이동 API 작성

완료 기준:

- 하나의 이미지를 열면 같은 폴더 목록이 생성된다.
- 좌/우 이동용 내부 API가 동작한다.
- 첫/마지막 경계 처리가 안정적이다.

현재 상태:

- 완료

### 4.8 Phase 8. OCR 및 온디바이스 번역

핵심 뷰어 흐름이 안정화된 뒤 후반부에 붙이는 기능이다.
이미지 표시 자체를 바꾸기보다, 현재 보고 있는 이미지에 대해 보조 분석 결과를 제공하는 방식이 적절하다.

구현 항목:

- 현재 이미지 기준 OCR 실행
- OCR 원문 결과 모델
- 대상 언어 선택 또는 시스템 언어 기반 번역
- 온디바이스 번역 실행
- 원문/번역문 결과 패널
- 텍스트 복사
- OCR/번역 실패 상태 처리

권장 작업 순서:

1. `ViewerState`와 분리된 `ImageTextAnalysisState` 또는 이에 준하는 상태 모델 정의
2. `Vision` 기반 OCR 서비스 추가
3. OCR 결과를 현재 `ImageItem`에 연결
4. `Translation` 기반 번역 서비스 추가
5. 결과 패널 UI 연결
6. 이미지 전환 시 결과 유지/초기화 정책 확정

완료 기준:

- 사용자가 현재 이미지에서 OCR을 실행할 수 있다.
- 추출된 텍스트를 별도 패널에서 읽고 복사할 수 있다.
- 번역을 실행하면 원문과 번역문이 함께 표시된다.
- 텍스트가 없거나 번역이 불가능한 경우 이해 가능한 오류가 표시된다.

현재 상태:

- 미시작

## 7. 기술 메모

### 7.1 권장 구조

- `App` 또는 앱 델리게이트 브리지: 파일 열기 이벤트
- `ViewerState`: 현재 이미지, 컬렉션, 보기 상태 관리
- `ImageCollectionProvider`: 폴더/압축 목록 제공
- `ImageLoader`: 이미지 로딩
- `ViewerScreen`: 메인 UI
- `TextRecognitionService`: 현재 이미지 OCR
- `ImageTranslationService`: OCR 결과 번역

### 7.2 먼저 분리해두면 좋은 상태

- 현재 파일 URL
- 현재 컬렉션
- 현재 인덱스
- 보기 모드
- 확대 배율
- 회전/반전 상태
- 툴바 표시 여부
- 정보 오버레이 표시 여부
- 현재 이미지별 OCR 결과
- 번역 대상 언어 및 번역 결과

### 7.3 테스트 우선 항목

- 폴더 정렬이 기대대로 되는지
- 첫/마지막 이미지 경계 처리
- 지원하지 않는 파일 처리
- 손상된 이미지 처리
- 큰 이미지 로딩 중 UI 멈춤 여부
- ZIP 내부 이미지 순서와 탐색
- OCR 결과 언어 품질과 긴 텍스트 처리
- 번역 가능 언어쌍 및 실패 처리

## 8. MVP 정의

아래를 만족하면 1차 MVP로 본다.

- 이미지 파일을 열 수 있다.
- 같은 폴더 이미지를 좌/우로 넘길 수 있다.
- 창 맞춤과 원본 크기 보기가 된다.
- 확대/축소가 된다.
- 전체화면이 된다.
- 툴바와 정보 표시가 기본적으로 동작한다.
- ZIP 파일 내부 이미지를 순서대로 볼 수 있다.

OCR과 번역은 MVP 이후 후반부 기능으로 두고, ZIP과 기본 상호작용이 정리된 뒤 붙이는 것을 권장한다.

## 9. 현재 추천 착수 순서

바로 구현을 시작한다면 다음 3개를 먼저 한다.

1. 앱 상태 모델과 파일 열기 경로 만들기
2. 단일 이미지 표시 + 폴더 기반 탐색 만들기
3. 메인 캔버스와 최소 툴바 붙이기

이 3개가 되면 이후 기능은 대부분 그 위에 안정적으로 쌓을 수 있다.

## 10. 다음 세션 시작 포인트

다음 개발 세션에서는 아래 순서로 이어가는 것을 권장한다.

1. 확대 상태에서 드래그 패닝 정책 구현
2. 더블클릭 동작을 `Fit` / `Actual` 전환으로 연결
3. 정보 오버레이 `항상 표시` / `자동 숨김` 토글 추가
4. ZIP 기반 `ArchiveImageCollection` 설계 시작
5. OCR 상태 모델과 `Vision`/`Translation` 적용 방식 정리

현재는 일반 폴더 기반 탐색과 보기 상태 구조가 안정화된 상태이므로, 먼저 압축 파일 지원과 입력 UX를 정리한 뒤 OCR/번역 기능을 얹는 순서가 적절하다.
