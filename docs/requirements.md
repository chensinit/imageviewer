# 이미지 뷰어 요구사항 문서

## 1. 문서 목적

이 문서는 macOS용 Xcode 프로젝트 기반 이미지 뷰어 애플리케이션의 1차 요구사항을 정의한다.
목표는 꿀뷰와 유사한 사용 경험을 가진 가벼운 이미지 뷰어를 만드는 것이다.

핵심 목표는 다음과 같다.

- 일반 이미지 파일을 빠르게 연다.
- 다양한 화면 맞춤 방식과 스케일 방식을 지원한다.
- Finder에서 파일 드래그 앤 드롭으로 이미지를 연다.
- 기본 앱으로 지정했을 때 이미지 파일을 더블클릭하면 바로 열린다.
- 같은 폴더 내 이미지들을 키보드로 연속 탐색한다.
- 압축 파일 내부 이미지까지 순서대로 탐색한다.
- 이미지 안의 텍스트를 온디바이스 OCR로 추출하고 번역 결과를 확인한다.

## 2. 제품 개요

앱은 macOS 데스크톱 환경에서 동작하는 단일 이미지 뷰어이다.
이미지 렌더링 자체는 OS 또는 프레임워크의 기본 제공 기능을 최대한 활용한다.
앱은 복잡한 편집 기능보다 "빠른 열기, 자연스러운 탐색, 다양한 표시 옵션"에 집중한다.

## 3. 범위

### 3.1 1차 범위

- 로컬 이미지 파일 열기
- 드래그 앤 드롭으로 파일 열기
- Finder 기본 앱 연동
- 같은 폴더 내 다음/이전 이미지 탐색
- 압축 파일 열기 및 내부 이미지 탐색
- 확대/축소 표시
- 화면 맞춤 방식 변경
- 스케일 방식 변경
- 전체화면 전환
- 기본 정보 표시
- 회전 및 반전
- 마우스 기반 탐색 및 확대/축소
- 최근 파일/최근 항목
- 기본적인 오류 처리
- 현재 이미지 기준 OCR 텍스트 인식
- 인식된 텍스트의 온디바이스 번역

### 3.2 1차 범위 제외

- 이미지 편집 기능
- 슬라이드쇼
- EXIF 편집
- 썸네일 브라우저
- 다중 창 관리 고도화
- 클라우드 스토리지 직접 연동

## 4. 사용자 시나리오

### 4.1 파일 직접 열기

사용자는 이미지 파일을 더블클릭하거나 앱에 전달하여 바로 이미지를 본다.

### 4.2 드래그 앤 드롭

사용자는 Finder에서 이미지 파일 또는 압축 파일을 앱 창으로 드래그하여 내용을 연다.

### 4.3 폴더 내 연속 탐색

사용자가 하나의 이미지를 열면, 앱은 같은 폴더의 이미지 목록을 구성하고 왼쪽/오른쪽 방향키로 이전/다음 이미지를 보여준다.

### 4.4 압축 파일 탐색

사용자가 ZIP 등 지원 압축 파일을 열면, 앱은 압축 내부의 이미지 목록을 순서대로 구성하고 일반 폴더와 유사하게 탐색할 수 있게 한다.

### 4.5 보기 옵션 변경

사용자는 이미지가 표시되는 동안 스케일 방식과 화면 채우기 방식을 바꿔 원하는 품질과 레이아웃으로 본다.

### 4.6 빠른 정보 확인

사용자는 현재 이미지의 파일명, 해상도, 파일 크기 등 기본 정보를 바로 확인할 수 있다.

### 4.7 마우스 조작

사용자는 마우스 휠이나 더블클릭으로 이미지 탐색, 확대/축소, 화면 전환을 수행할 수 있다.

### 4.8 이미지 텍스트 번역

사용자는 텍스트가 포함된 이미지를 보고 있을 때 OCR을 실행해 텍스트를 추출하고, Apple이 제공하는 온디바이스 번역 기능으로 결과를 확인한다.

## 5. 기능 요구사항

### 5.1 파일 열기

- 앱은 이미지 파일을 직접 열 수 있어야 한다.
- 앱은 Finder의 "다음으로 열기" 및 기본 앱 지정 대상으로 노출되어야 한다.
- 사용자가 이미지 파일을 더블클릭해 앱이 실행된 경우 해당 이미지를 즉시 표시해야 한다.
- 앱이 이미 실행 중인 상태에서 다른 이미지 파일 열기 요청을 받으면 현재 창 또는 지정된 정책에 따라 해당 이미지를 표시해야 한다.

### 5.2 드래그 앤 드롭

- 앱 메인 창은 파일 드래그 앤 드롭을 지원해야 한다.
- 드롭 가능한 대상:
  - 단일 이미지 파일
  - 복수 이미지 파일
  - 단일 압축 파일
- 복수 이미지 파일이 드롭된 경우 표시 순서를 정의해야 한다.
- 지원하지 않는 파일이 드롭되면 사용자에게 열 수 없음을 알려야 한다.

### 5.3 이미지 표시

- 앱은 선택한 이미지를 메인 뷰에 표시해야 한다.
- 이미지 표시에는 시스템 기본 이미지 렌더링 기능을 우선 사용한다.
- 앱은 큰 이미지도 가능한 한 빠르게 첫 화면을 표시해야 한다.
- 로딩 중에는 빈 화면 대신 최소한의 로딩 상태 또는 이전 화면 유지 정책을 정의해야 한다.

### 5.4 스케일 방식

앱은 최소한 다음 스케일 방식을 지원해야 한다.

- Nearest Neighbor 또는 이에 준하는 저보간 방식
- Linear 또는 이에 준하는 일반 보간 방식
- 필요 시 향후 고품질 보간 방식 확장 가능 구조

세부 요구사항:

- 사용자는 메뉴 또는 UI 컨트롤로 스케일 방식을 변경할 수 있어야 한다.
- 스케일 방식 변경 시 현재 표시 중인 이미지에 즉시 반영되어야 한다.
- 픽셀 아트 성격의 이미지에서는 저보간 방식 사용이 가능해야 한다.

### 5.5 화면 채우기 방식

앱은 최소한 다음 표시 방식을 지원해야 한다.

- 원본 크기
- 창 크기에 맞춤
- 비율 유지 맞춤
- 화면 채우기
- 늘리기 또는 비율 무시 채우기

세부 요구사항:

- 사용자는 메뉴 또는 단축키로 표시 방식을 전환할 수 있어야 한다.
- 비율 유지 모드에서는 원본 종횡비가 유지되어야 한다.
- 화면 채우기 모드에서는 일부 영역이 잘릴 수 있다.
- 원본 크기 모드에서는 스크롤 또는 패닝 정책이 필요하다.

### 5.6 확대/축소 및 이동

- 사용자는 확대/축소를 수행할 수 있어야 한다.
- 확대 상태에서 이미지 이동이 가능해야 한다.
- 화면 맞춤 모드로 복귀하는 방법이 있어야 한다.
- 확대 배율 정책의 최소/최대 범위를 정의해야 한다.

참고:
확대/축소는 현재 사용자 요청의 핵심 항목은 아니지만 실제 사용성을 위해 1차 범위에 포함한다.

### 5.7 폴더 기반 탐색

- 단일 이미지 파일을 열면 앱은 같은 폴더의 지원 이미지 목록을 수집해야 한다.
- 사용자는 키보드 좌/우 방향키로 이전/다음 이미지를 탐색할 수 있어야 한다.
- 정렬 기준은 Finder와 유사한 파일명 기준 정렬을 기본값으로 한다.
- 첫 이미지에서 이전으로 이동하거나 마지막 이미지에서 다음으로 이동할 때의 정책을 정의해야 한다.

권장 기본 정책:

- 순환 이동 없음
- 경계에서 입력 시 변화 없음

### 5.8 압축 파일 지원

- 앱은 최소한 ZIP 형식을 1차 지원 대상으로 삼는다.
- 압축 파일을 열면 내부 항목 중 이미지 파일만 추출 또는 스트리밍 대상으로 구성해야 한다.
- 내부 이미지 목록은 폴더 경로를 포함한 안정적인 정렬 기준으로 순서가 결정되어야 한다.
- 사용자는 일반 폴더 탐색과 동일하게 좌/우 방향키로 이전/다음 이미지를 볼 수 있어야 한다.
- 압축 해제 실패, 비밀번호 필요, 손상 파일 등 예외 상황을 사용자에게 알려야 한다.
- 대용량 압축 파일에 대해 전체 선해제를 강제하지 않도록 설계해야 한다.

### 5.9 지원 포맷

초기 지원 권장 포맷:

- PNG
- JPEG/JPG
- GIF
- BMP
- TIFF
- WebP
- HEIC/HEIF

압축 파일 초기 지원 권장 포맷:

- ZIP

후속 검토 포맷:

- 7z
- RAR
- CBZ

주의:
실제 지원 가능 포맷은 macOS 기본 디코더와 선택 라이브러리 제약을 반영해 확정해야 한다.

### 5.10 키보드 조작

최소 요구 키:

- Left Arrow: 이전 이미지
- Right Arrow: 다음 이미지
- Home: 첫 이미지 후보
- End: 마지막 이미지 후보
- Escape: 기본 배율 또는 기본 보기 상태 복귀 후보
- Enter 또는 Command + Control + F: 전체화면 전환 후보
- Command + O: 파일 열기 후보
- Command + Plus/Minus: 확대/축소 후보

권장 추가 키:

- `0`: 원본 크기 또는 100% 배율
- `1`: 창 맞춤
- `R`: 시계 방향 회전
- `L`: 반시계 방향 회전
- `H`: 좌우 반전 후보

세부 단축키는 구현 단계에서 최종 확정한다.

### 5.11 마우스 조작

- 마우스 휠로 이전/다음 이미지 탐색을 지원해야 한다.
- 수정 키와 함께 마우스 휠 사용 시 확대/축소를 지원해야 한다.
- 더블클릭으로 창 맞춤과 원본 크기 전환 또는 최대화 전환 중 하나를 수행할 수 있어야 한다.
- 확대 상태에서 드래그로 이미지 이동을 지원해야 한다.

### 5.12 전체화면 및 창 동작

- 사용자는 전체화면 전환을 수행할 수 있어야 한다.
- 전체화면 상태에서도 키보드 탐색과 확대/축소가 가능해야 한다.
- 창 크기 변경 시 현재 보기 모드 정책에 따라 이미지가 즉시 다시 배치되어야 한다.

### 5.13 이미지 회전 및 반전

- 사용자는 현재 이미지를 화면 표시 기준으로 회전할 수 있어야 한다.
- 최소한 90도 단위 회전을 지원해야 한다.
- 사용자는 좌우 반전을 수행할 수 있어야 한다.
- 회전 및 반전은 원본 파일을 수정하지 않는 뷰 레벨 동작이어야 한다.
- 다음 이미지로 이동 시 회전 상태 유지 여부를 정책으로 정의해야 한다.

권장 기본 정책:

- 회전 및 반전 상태는 현재 이미지에만 적용
- 다른 이미지로 이동하면 기본 상태로 복귀

### 5.14 기본 정보 표시

- 앱은 현재 이미지의 기본 정보를 표시할 수 있어야 한다.
- 최소 표시 항목:
  - 파일명
  - 픽셀 크기
  - 파일 크기
  - 현재 순번과 전체 개수
- 정보 표시는 토글 가능해야 한다.
- EXIF 전체 편집은 범위 밖이지만, 읽기 전용 기본 메타데이터 노출은 허용한다.

### 5.15 최근 파일/최근 항목

- 앱은 최근에 연 파일 목록을 유지할 수 있어야 한다.
- 사용자는 메뉴를 통해 최근 파일을 다시 열 수 있어야 한다.
- 최근 항목이 삭제되었거나 이동된 경우 안전하게 실패해야 한다.
- 사용자는 최근 파일 메뉴를 비울 수 있어야 한다.

### 5.15.1 현재 컬렉션 목록 보기

- 앱은 현재 탐색 중인 이미지 컬렉션의 목록을 별도 UI로 보여줄 수 있어야 한다.
- 일반 이미지 열기 상태에서는 같은 폴더 내 이미지 목록을 보여줘야 한다.
- ZIP 열기 상태에서는 압축 내부 이미지 목록을 보여줘야 한다.
- 사용자는 목록에서 항목을 선택해 즉시 해당 이미지로 이동할 수 있어야 한다.
- 현재 표시 중인 항목은 목록에서 식별 가능해야 한다.

### 5.16 오류 처리

앱은 다음 상황에서 사용자에게 명확한 피드백을 제공해야 한다.

- 지원하지 않는 파일 형식
- 파일 접근 불가
- 손상된 이미지
- 압축 해제 실패
- 압축 파일 내부에 이미지가 없음

오류 메시지는 기술적인 내부 예외를 그대로 노출하기보다 사용자가 이해할 수 있는 문장이어야 한다.

### 5.17 OCR 텍스트 추출

- 앱은 현재 표시 중인 이미지에 대해 OCR 실행 기능을 제공해야 한다.
- OCR은 Apple이 제공하는 온디바이스 텍스트 인식 기능을 우선 사용해야 한다.
- OCR은 현재 보이는 이미지를 기준으로 실행해야 하며, 원본 파일을 외부 서버로 업로드하지 않아야 한다.
- 텍스트가 감지되지 않은 경우 사용자에게 결과가 없음을 알려야 한다.
- OCR 실행 중에는 진행 중 상태를 간단히 표시해야 한다.

### 5.18 번역

- 앱은 OCR로 추출한 텍스트를 번역할 수 있어야 한다.
- 번역은 Apple이 제공하는 온디바이스 번역 기능을 우선 사용해야 한다.
- 번역 방향은 시스템 언어 또는 사용자가 선택한 대상 언어를 기준으로 결정할 수 있어야 한다.
- 사용자는 원문과 번역문을 함께 확인할 수 있어야 한다.
- 번역 불가 언어, 모델 미지원, 기기 제한 등으로 실패한 경우 사용자에게 이유를 설명해야 한다.

### 5.19 OCR/번역 결과 표시

- OCR 및 번역 결과는 메인 캔버스를 과도하게 가리지 않는 별도 패널 또는 시트로 제공하는 것을 기본 정책으로 한다.
- 필요 시 이미지 위 텍스트 영역 하이라이트를 후속 확장으로 고려할 수 있어야 한다.
- 사용자는 결과 텍스트를 복사할 수 있어야 한다.
- 현재 이미지가 바뀌면 이전 OCR/번역 결과를 재사용할지 초기화할지 정책을 정의해야 한다.

## 6. 비기능 요구사항

### 6.1 성능

- 일반적인 이미지 열기 체감 속도는 즉각적이어야 한다.
- 같은 폴더 또는 같은 압축 파일 내 다음 이미지 전환은 가능한 한 지연이 적어야 한다.
- 대용량 이미지 또는 대량 항목 압축 파일도 UI 멈춤 없이 처리해야 한다.

### 6.2 안정성

- 손상 파일 또는 비정상 압축 파일로 인해 앱이 종료되면 안 된다.
- 파일이 없거나 접근 권한이 없을 때 안전하게 실패해야 한다.

### 6.3 확장성

- 이미지 소스는 "일반 폴더"와 "압축 파일 내부"를 같은 탐색 모델로 다룰 수 있어야 한다.
- 스케일 방식, 지원 포맷, 압축 포맷이 향후 추가 가능하도록 구조화해야 한다.

### 6.4 사용성

- 첫 실행 시에도 즉시 사용할 수 있을 만큼 UI가 단순해야 한다.
- 마우스 없이도 기본 탐색이 가능해야 한다.
- 현재 보기 모드가 사용자가 인지 가능해야 한다.
- OCR/번역 기능은 이미지 보기 흐름을 방해하지 않는 보조 기능이어야 한다.

## 7. 시스템 설계 방향

### 7.1 이미지 소스 추상화

다음 두 소스를 공통 인터페이스로 다루는 구조를 권장한다.

- 로컬 폴더 기반 이미지 시퀀스
- 압축 파일 기반 이미지 시퀀스

예시 개념:

- `ImageItem`: 실제 표시 가능한 단일 이미지 단위
- `ImageCollection`: 순서가 있는 이미지 목록
- `ImageSource`: 파일 시스템 또는 압축 파일 같은 원천

### 7.2 정렬 정책

- 기본 정렬은 파일명 기준 자연 정렬을 사용한다.
- 압축 파일 내부도 동일한 정렬 기준을 적용한다.
- 숨김 파일, 메타 파일, 디렉터리는 표시 대상에서 제외한다.

### 7.3 캐싱 정책

- 현재 이미지
- 이전 이미지
- 다음 이미지

정도는 선로드 또는 캐시를 고려할 수 있다.
단, 메모리 사용량이 과도해지지 않도록 제한이 필요하다.

### 7.4 OCR/번역 파이프라인

- 이미지 표시와 OCR/번역 실행은 분리된 비동기 작업으로 다루는 것을 권장한다.
- OCR 결과는 현재 이미지 식별자와 연결해 임시 캐시할 수 있어야 한다.
- 번역은 OCR 원문을 입력으로 사용하고, 언어쌍별로 결과를 분리해 저장할 수 있어야 한다.
- 앱 기본 정책은 네트워크 의존 없는 온디바이스 처리로 둔다.

## 8. UI 요구사항

### 8.1 기본 화면

- 이미지를 크게 표시하는 단일 메인 캔버스
- 파일 드롭이 가능함을 알 수 있는 초기 상태
- 보기 방식 변경 메뉴 또는 툴바

### 8.2 상태 표시

필요 시 다음 정보 표시를 고려한다.

- 현재 파일명
- 전체 개수와 현재 순번
- 현재 확대 배율
- 현재 표시 방식
- 현재 스케일 방식
- 현재 이미지 해상도
- 현재 파일 크기
- OCR 실행 상태
- 번역 대상 언어

### 8.3 빈 상태

- 아직 파일이 열리지 않은 상태에서 드래그 앤 드롭 안내를 제공한다.

## 9. 앱 연동 요구사항

### 9.1 기본 앱 등록

- 앱 번들 설정에서 지원 이미지 포맷 및 지원 압축 포맷에 대한 문서 타입 등록이 가능해야 한다.
- 사용자가 Finder에서 해당 파일 형식의 기본 앱으로 지정할 수 있어야 한다.

### 9.2 파일 열기 이벤트 처리

- 앱 시작 시 전달된 파일 열기 요청 처리
- 앱 실행 중 추가 파일 열기 요청 처리
- 한 번에 여러 파일 전달 시 처리 정책 정의

권장 기본 정책:

- 마지막으로 선택된 파일을 메인 표시 대상으로 사용
- 전달된 파일이 같은 폴더라면 해당 집합 구성 가능

## 10. 예외 및 결정 필요 항목

아래 항목은 구현 전에 확정이 필요하다.

- 복수 파일 드롭 시 정렬 기준
- 여러 창 지원 여부
- 원본 크기 모드에서 스크롤 방식
- 애니메이션 GIF의 재생 여부 또는 첫 프레임만 표시 여부
- 압축 파일 지원 범위를 ZIP만으로 시작할지 여부
- 비밀번호 압축 파일 대응 범위
- 메타데이터 회전 정보(EXIF orientation) 처리 정책
- 마우스 휠의 기본 동작을 이미지 넘김과 스크롤 중 무엇으로 둘지 여부
- 더블클릭 동작을 원본 크기 전환과 전체화면 전환 중 무엇으로 둘지 여부

## 12. 수용 기준

다음 조건을 만족하면 1차 요구사항을 충족한 것으로 본다.

- 사용자가 PNG 또는 JPG 파일을 더블클릭했을 때 앱에서 이미지가 열린다.
- 사용자가 앱 창에 이미지 파일을 드래그하면 바로 표시된다.
- 사용자가 열린 이미지에서 좌/우 방향키를 누르면 같은 폴더 내 이전/다음 이미지로 이동한다.
- 사용자가 보기 모드와 스케일 방식을 바꾸면 현재 이미지에 즉시 반영된다.
- 사용자가 ZIP 파일을 열면 내부 이미지가 첫 화면에 표시되고 좌/우 방향키로 순차 탐색된다.
- 사용자가 전체화면 전환 후에도 동일하게 이미지 탐색과 보기 변경을 수행할 수 있다.
- 사용자가 정보 표시를 켜면 현재 파일명, 해상도, 순번이 보인다.
- 사용자가 회전 또는 반전을 수행하면 현재 표시 이미지에 즉시 반영된다.
- 사용자가 OCR을 실행하면 현재 이미지에서 추출된 텍스트를 확인할 수 있다.
- 사용자가 번역을 실행하면 OCR 원문과 번역문을 함께 확인할 수 있다.
- 지원하지 않는 파일 또는 손상 파일을 열었을 때 앱이 종료되지 않고 오류를 표시한다.

## 13. 구현 메모

현재 단계에서는 요구사항 문서 작성이 목적이므로 구현 상세는 제한적으로만 기술한다.
다만 다음 방향을 권장한다.

- UI: SwiftUI 기반
- 파일 열기 이벤트: `App` 또는 `NSApplicationDelegate` 브리지 검토
- 이미지 표시: macOS 기본 이미지 디코딩 및 렌더링 우선
- 압축 처리: ZIP 우선 지원, 필요 시 라이브러리 도입 검토
- 탐색 모델: 폴더/압축 공통 시퀀스 모델로 통합
- OCR: `Vision` 기반 온디바이스 텍스트 인식 우선 검토
- 번역: `Translation` 프레임워크 기반 온디바이스 번역 우선 검토
